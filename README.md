# HTML & CSS Learning Repository

## Overview
This repository contains a collection of structured tasks designed to build a strong foundation in HTML and CSS.  
The exercises progress from fundamental layout techniques to advanced state-driven UI components implemented purely with CSS.

The primary objective of this module is to develop proficiency in:

- Semantic HTML structure
- Responsive design principles
- Layout engineering using Flexbox and Grid
- Advanced CSS selectors and pseudo-classes
- Pure CSS interactivity without JavaScript
- Professional code organization and documentation practices

---

## Topics Covered

### Fundamentals
- Semantic webpage layout using HTML5 elements
- Typography, spacing, and color styling
- Responsive design using media queries

### Layout Techniques
- CSS Grid for macro layout systems
- Flexbox for alignment and component-level layout
- Sticky positioning and overlapping elements using z-index

### UI Components (Pure CSS)
- Card components with hover effects
- Responsive navigation dropdown
- Modal popup using `:target`
- Tabbed interface using `:checked`
- Accordion using checkbox hack
- Carousel/slider using keyframe animations
- Multi-page navigation simulation using `:target`

### Advanced CSS Concepts
- State management using pseudo-classes
- CSS animation and transition systems
- DOM order dependency in CSS selectors
- Responsive reordering of layout elements
- CSS-only routing simulation

---

## Repository Structure

Each task follows the structure:

html-css/task-x/

Each task includes:

- `index.html`
- `styles.css`
- `README.md`

This ensures modularity, clarity, and ease of review.

---

## How to Run

1. Navigate to any task folder.
2. Open `index.html` in a web browser.
3. Resize the viewport to test responsive behavior.

---

## Learning Outcomes

After completing this module, the following competencies are developed:

- Ability to design responsive web layouts
- Understanding of CSS layout systems (Grid vs Flexbox)
- Capability to build interactive UI components without JavaScript
- Strong grasp of CSS selector logic and state-driven UI patterns
- Professional repository organization and documentation skills

---

## Notes

- No frontend frameworks were used (React, Angular, etc.)
- Only essential CSS techniques were applied to ensure conceptual clarity
- All implementations prioritize readability, maintainability, and industry-relevant practices
